# 設定と操作
このページではインスペクター上から設定できる項目を解説します。


## マテリアル共通設定
マテリアルの値を上書きします。<br />
ニアクリップの強制やライトカラーモードなどを一括ですべてのマテリアルに反映します。<br />
マテリアル値の上書きのみでアニメーションでの制御はされません。
:::note

`ニヤクリップの強制`や`ワールドAOブロッカー`などをすべてのマテリアルに一括で設定したい時に使用できます。

:::

## マテリアル設定

![mat-setting](/img/mat-setting.png)
ここで指定された`プロパティ`をアニメーションで制御します。<br />
カテゴリーが無効の場合は子の`プロパティ`は**完全に無効**になります。<br />
無効化した場合PLCはその`プロパティ`に対して**一切の制御**を行いません。

### プロパティごとの制御項目

- `初期値` : 初期設定の値をプロパティの絶対値で指定します。
- `操作範囲(下限|上限)` : ExMenuで操作できる範囲をプロパティの絶対値で指定します。
- `アニメーション` : アニメーションで制御するかを指定します。
- `同期` : Expression Parameterを使用して同期するかを指定します。
- `保存` : アバター再読み込み時にExMenuの設定値を維持するかを指定します。

:::info

`初期値` `操作範囲`は`プロパティの絶対値`ですがExMenuでは `%` での表記になります。<br />
例 : `初期値 0.3` で `操作範囲 0-0.5`の場合はExMenuでので初期値は `60%` になります。

:::

## マテリアルについて

### マテリアルの値の上書き
PLCではアニメーションだけではなくマテリアルに直接、値を上書きする処理があります。<br />
上書き処理は以下の順番で実行されます。

1. `アニメーションされてるプロパティの初期値`
2. `マテリアル共通設定での一括設定`
3. `アニメーションのチェックが外されたプロパティ`

:::info

マテリアルの上書き処理は、前の処理が後の処理に上書きされて優先されます。

:::

#### アニメーションされてるプロパティの初期値
AAOとの互換性の問題でアニメーションが有効のプロパティは設定した初期値がマテリアルに書き込まれます。

#### マテリアル共通設定での一括設定
マテリアル共通設定での設定値がすべてのマテリアルに一括で書き込まれます。

#### アニメーションのチェックが外されたプロパティ
アニメーションのチェックが外れている場合は`ユーザーが指定した初期値`に**固定したい**と解釈します。<br />
アニメーションクリップやコントローラーは生成されずに、**マテリアルの値を上書きする処理**のみが行われます。

### マテリアルのLockについて

マテリアルはビルド時にCloneされ、アップロード時にすべて自動でLockされる仕様です。<br />
そのため、事前にLockする必要はありません。<br />
すでにLockされているマテリアルも問題なく動作します。
